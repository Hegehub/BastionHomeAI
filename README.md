# Bastion Home AI

**Bastion Home AI** — локальная суверенная ИИ-платформа для умного дома, работающая на внутреннем домашнем сервере.  
Проект объединяет голосовое управление, камеры, жесты, датчики, роли доступа, автоматизацию, локальную память, безопасность, приватность и функции заботы для пожилых людей и людей с ограниченными возможностями.

Главная идея проекта:

> Дом должен быть не просто умным, а безопасным, приватным, адаптивным и управляемым владельцем.

## О проекте

**Bastion Home AI** — это домашний управляющий ИИ-агент, который размещается на локальном сервере внутри дома и взаимодействует с пользователями через:

- голосовые команды;
- камеры;
- микрофоны;
- динамики;
- жесты;
- датчики;
- умные устройства;
- локальную сеть;
- домашнюю автоматизацию.

Система предназначена для управления домом, помощи жильцам, повышения безопасности, контроля доступа, адаптации среды под конкретного человека и сохранения приватности.

## Ключевая концепция

Обычный умный дом работает по простой схеме:

```text
Команда → Устройство
````

**Bastion Home AI** работает иначе:

```text
Человек
  ↓
Идентификация
  ↓
Роль
  ↓
Контекст
  ↓
Намерение
  ↓
Оценка риска
  ↓
Policy Engine
  ↓
Исполнение
  ↓
Audit Log
  ↓
Объяснение
```

ИИ не должен напрямую управлять критическими действиями.
Он определяет намерение пользователя, а решение о выполнении принимает отдельный слой правил и безопасности.

## Основные принципы

### 1. Локальность

Критически важные функции должны работать внутри дома:

* свет;
* датчики;
* безопасность;
* роли;
* голосовые команды;
* базовая автоматизация;
* тревожные сценарии;
* журнал событий.

---

### 2. Приватность

Видео, голос, события дома, лица, расписания и привычки пользователей не должны передаваться во внешние сервисы без явного разрешения владельца.

---

### 3. Владелец — верховный контролёр

Система не является самостоятельным владельцем дома.

```text
Owner > Policy Engine > Agent > Executor > Devices
```

---

### 4. Безопасная автономность

ИИ может помогать, предлагать и автоматизировать, но опасные действия требуют проверки, роли, контекста и подтверждения.

---

### 5. Объяснимость

Система должна уметь объяснить:

* почему действие выполнено;
* почему действие заблокировано;
* почему включена тревога;
* почему требуется подтверждение;
* какое правило было применено.

### Identity Engine

Модуль идентификации.

Отвечает за:

* распознавание пользователя;
* определение роли;
* проверку доверия;
* сопоставление лица, голоса и устройства;
* работу с гостями;
* временный доступ.

---

### Context Engine

Модуль понимания ситуации.

Учитывает:

* комнату;
* время;
* режим дома;
* людей рядом;
* открытые двери и окна;
* температуру;
* освещение;
* уровень шума;
* наличие гостей;
* активные устройства;
* тревожные события;
* профиль пользователя.

---

### Policy Engine

Центральный модуль безопасности.

Он решает:

* разрешено ли действие;
* нужен ли второй фактор;
* нужен ли владелец;
* нужно ли подтверждение;
* можно ли выполнить команду в текущем режиме;
* нужно ли записать действие в журнал.

Пример политики:

```yaml
rule_id: unlock_front_door
description: "Открытие входной двери"
risk_level: high

allowed_roles:
  - owner
  - admin

required_factors:
  - face_match
  - voice_match
  - trusted_device_nearby

confirmation_required: true
audit_required: true

deny_if:
  - house_mode == "away"
  - unknown_person_nearby == true
```

---

### Agent Brain

ИИ-слой системы.

Назначение:

* понимать естественные фразы;
* определять намерение;
* строить план действий;
* объяснять решения;
* вести диалог;
* использовать локальную память;
* работать с RAG;
* помогать пользователю управлять домом.

Важно:

> LLM не должна напрямую управлять устройствами.
> Все действия проходят через Policy Engine и Executor.

---

### Action Executor

Слой исполнения команд.

Он подключается к:

* Home Assistant;
* MQTT;
* Zigbee2MQTT;
* Z-Wave;
* Matter;
* Thread;
* ESPHome;
* умным лампам;
* датчикам;
* замкам;
* шторам;
* климату;
* камерам;
* колонкам.

Executor выполняет только разрешённые команды.

---

### Audit Log

Журнал событий.

Фиксирует:

* кто дал команду;
* через какой интерфейс;
* какое действие было запрошено;
* какое правило было применено;
* действие разрешено или отклонено;
* какие устройства были задействованы;
* было ли подтверждение;
* почему система приняла решение.

Пример записи:

```json
{
  "time": "2026-05-15T21:30:00",
  "actor": "owner",
  "source": "voice",
  "intent": "activate_night_mode",
  "decision": "allowed",
  "policy": "night_mode_owner_allowed",
  "actions": [
    "lock_doors",
    "turn_off_lights",
    "arm_security"
  ],
  "explanation": "Owner requested night mode. All actions are allowed by policy."
}
```

### Household Anomaly Detection

Обнаружение бытовых аномалий без медицинской диагностики.

Примеры:

* холодильник не открывался весь день;
* ванная занята необычно долго;
* нет активности дольше обычного.

Система не ставит диагнозы.
Она фиксирует бытовые риски и предлагает проверить ситуацию.

## Безопасность

Критические действия запрещено выполнять только по одному фактору.

Критические действия:

* открыть входную дверь;
* отключить охрану;
* изменить права пользователя;
* удалить журналы;
* отключить камеры;
* дать гостю доступ;
* открыть гараж;
* отключить датчики дыма, газа или воды.

Для таких действий требуются:

```text
роль владельца
многофакторная проверка
контекстная проверка
подтверждение
audit log
```

## Предупреждение

Bastion Home AI взаимодействует с физическим пространством дома.
Поэтому любые функции, связанные с дверями, замками, охраной, камерами, пожилыми людьми, детьми и аварийными ситуациями, должны проектироваться с повышенным вниманием к безопасности, приватности и юридической ответственности.

Система не должна использоваться как медицинское устройство и не должна ставить диагнозы.

## Краткая формула

**Bastion Home AI** — это локальная домашняя ИИ-платформа, которая управляет домом, соблюдает роли доступа, защищает приватность, объясняет свои действия и адаптирует среду под человека.

Цель проекта:

> Сделать дом безопасным, понятным, автономным, доступным и действительно помогающим человеку.

```
```
